# B-tree Index Study

## Описание
B-tree (Balanced Tree) - наиболее универсальный и часто используемый тип индекса в PostgreSQL. Оптимален для диапазонных запросов, операторов сравнения (=, <, >, <=, >=, BETWEEN) и сортировки.

## Преимущества
- Поддерживает диапазонные запросы
- Эффективен для сортировки (ORDER BY)
- Работает с различными типами данных
- Поддерживает составные индексы

## Ограничения
- Больший размер по сравнению с некоторыми специализированными индексами
- Не оптимален для точечного поиска по хешу

## Сценарий использования
Тестирование производительности для:
1. Диапазонных запросов по возрасту и зарплате
2. Сортировки результатов
3. Временных диапазонов
4. JOIN операций с сортировкой

## Структура данных
- **users**: 500,000 записей с демографическими и финансовыми данными
- **orders**: 1,000,000 записей с информацией о заказах

## Созданные индексы
1. `idx_users_age` - возраст пользователей
2. `idx_users_salary` - зарплата для сортировки
3. `idx_users_created_date` - дата создания
4. `idx_users_age_salary` - составной индекс (возраст+зарплата)
5. `idx_orders_amount` - сумма заказа
6. `idx_orders_date` - дата заказа
7. `idx_orders_user_date` - составной индекс (пользователь+дата)

## Ожидаемые результаты
- Значительное ускорение диапазонных запросов (10-100x)
- Эффективное выполнение сортировки без временных файлов
- Оптимизация JOIN операций
