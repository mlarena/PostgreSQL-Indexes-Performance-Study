# Hash Index Study

## Описание
Hash индекс использует хеш-таблицу для быстрого точечного поиска по равенству. В PostgreSQL Hash индексы эффективны для операторов `=` (равенство), но не поддерживают диапазонные запросы или сортировку.

## Преимущества
- Очень быстрый точечный поиск по равенству
- Меньший размер по сравнению с B-tree для некоторых сценариев
- Эффективен для уникальных идентификаторов и кодов

## Ограничения
- Не поддерживает диапазонные запросы (<, >, BETWEEN)
- Не поддерживает сортировку (ORDER BY)
- Не поддерживает составные индексы
- Требует перестроения при изменении данных
- Не поддерживает частичные индексы

## Сценарий использования
Тестирование производительности для:
1. Поиска продуктов по уникальному коду
2. Поиска сессий по идентификатору
3. Поиска конфигурационных значений по ключу
4. Сравнение с B-tree индексом для точечного поиска

## Структура данных
- **products**: 300,000 записей с уникальными кодами продуктов
- **user_sessions**: 500,000 записей с хешированными ID сессий
- **config**: 15 записей конфигурационных параметров

## Созданные индексы
1. `idx_products_code_hash` - Hash индекс на коде продукта
2. `idx_sessions_id_hash` - Hash индекс на ID сессии
3. `idx_config_key_hash` - Hash индекс на ключе конфигурации
4. `idx_products_code_btree` - B-tree индекс для сравнения
5. `idx_products_category_hash` - Hash индекс на категории (низкая кардинальность)

## Ожидаемые результаты
- Hash индекс быстрее B-tree для точечного поиска по равенству
- B-tree более универсален (поддерживает диапазоны и сортировку)
- Hash индекс неэффективен для низкокардинальных данных
