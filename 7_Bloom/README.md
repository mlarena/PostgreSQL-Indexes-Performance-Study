# Bloom Index Study

## Описание
Bloom индекс использует фильтр Блума для эффективной многоколоночной фильтрации с условиями OR. Оптимален для запросов с множеством условий по разным колонкам.

## Преимущества
- Эффективен для многоколоночных запросов с OR
- Компактный размер по сравнению с множеством B-tree индексов
- Быстрый отсев нерелевантных записей
- Хорош для аналитических запросов с фильтрацией по многим атрибутам

## Ограничения
- Только для условий равенства (=)
- Не поддерживает диапазонные запросы (<, >, BETWEEN)
- Может возвращать ложные срабатывания (требует дополнительной проверки)
- Не подходит для уникальных индексов

## Сценарий использования
Тестирование производительности для:
1. Многокритериального поиска пользователей
2. Фильтрации продуктов по различным атрибутам
3. Поиска заказов по статусам и методам
4. Анализа логов безопасности
5. Управления инвентарем по множеству критериев

## Структура данных
- **users**: 100,000 записей с атрибутами пользователей
- **products**: 80,000 записей продуктов с различными характеристиками
- **orders**: 150,000 записей заказов с разными статусами
- **security_logs**: 200,000 записей логов безопасности
- **inventory**: 120,000 записей инвентаря

## Созданные индексы

### Основные Bloom индексы:
1. `idx_users_bloom` - многокритериальный поиск пользователей
2. `idx_products_bloom` - фильтрация продуктов по атрибутам
3. `idx_orders_bloom` - поиск заказов по статусам и методам
4. `idx_security_logs_bloom` - анализ логов безопасности
5. `idx_inventory_bloom` - управление инвентарем

### Специализированные Bloom индексы:
- `idx_users_bloom_large` - с увеличенным размером
- `idx_products_numeric_bloom` - для числовых полей

### Для сравнения:
- `idx_users_username_btree` - B-tree для имен пользователей
- `idx_users_email_btree` - B-tree для email
- `idx_users_country_btree` - B-tree для стран
- `idx_products_brand_btree` - B-tree для брендов
- `idx_products_category_btree` - B-tree для категорий
- `idx_orders_status_btree` - B-tree для статусов заказов
- Составные B-tree индексы для сравнения

## Ожидаемые результаты
- Bloom эффективнее для запросов с множеством OR условий
- Значительно меньший размер по сравнению с множеством B-tree индексов
- Хорошая производительность для аналитических запросов
- Ложные срабатывания требуют дополнительной фильтрации
- Идеален для dashboards и систем поиска

## Рекомендации
- Используйте Bloom для запросов с 3+ условиями OR по разным колонкам
- Эффективен для колонок с высокой кардинальностью
- Рассмотрите для аналитических систем и поисковых интерфейсов
- Избегайте для уникальных индексов или точного поиска
- Настройте параметры длины и колонок для оптимизации
